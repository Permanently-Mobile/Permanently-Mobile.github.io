Process
Three lines of code.
Privacy for every message.
Integrate in minutes. No cloud dependencies, no data leaves your system unless you choose to send it.
your-app.js
import { PrivacyShield } from 'pm-privacy-guard';

const shield = new PrivacyShield({
  personalNames: ['John', 'Jane'],
  defaultMode: 'redact',
});

// Classify any message
const result = shield.classify("My SSN is 123-45-6789");
// { score: 0.95, categories: ['pii'], recommendation: 'privacy' }

// Full routing decision
const decision = shield.route(userMessage);
// { recommendedProvider: 'privacy', sanitized: { ... } }

FAQ
Common questions.
What AI providers does PM Privacy Guard work with? ▼
Any of them. PM Privacy Guard is provider-agnostic middleware. It classifies and sanitizes messages before they reach any provider -- Claude, OpenAI, Venice, Ollama, or your own models. You configure which providers are "privacy" (zero retention) and which are "standard."
Does PM Privacy Guard store my messages? ▼
No. The audit log stores SHA-256 hashes of messages, sensitivity scores, and routing decisions -- never the actual message content. Token maps (for reversible tokenization) are held in memory only and never persisted to disk.
Is this a cloud service? ▼
No. PM Privacy Guard is a self-hosted Node.js module that runs on your machine. Your data never leaves your infrastructure unless you choose to send it to an AI provider -- and PM Privacy Guard helps you control exactly what gets sent.
Is there a monthly fee? ▼
No. One-time payment. You own the software. No subscriptions, no usage limits beyond your tier's provider and feature counts.
Can I add custom detection patterns? ▼
Yes, with the Pro tier. Pass custom regex patterns per category in your config. Industry-specific terms, internal identifiers, proprietary data formats -- whatever your business needs to detect.
What about HIPAA / SOC2 / GDPR compliance? ▼
PM Privacy Guard provides the technical controls: PII detection, data sanitization, audit trails, and routing enforcement. These are building blocks for compliance -- but compliance also requires organizational policies, training, and sometimes third-party certification that are outside the scope of any software tool.
